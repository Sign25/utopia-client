# UI-инструкция: отображение роста ТКАНЕЙ Адама (§10.8)

Адам растит мозг при жизни в ДВЕ стадии. UI уже показывает первую (связи) —
эта инструкция добавляет вторую (ткани).

| Стадия | Что растёт | Где в графе | Метрики (уже/новые) |
|--------|-----------|-------------|---------------------|
| 1. Связи | рёбра {роль}→cerebellum | в графе организма | `topo_active`, `growth_kept`, `growth_reverted` (есть) |
| 2. **Ткани** | predictor-сайдкары | **ВНЕ графа** (изолированы от мотора) | `tissue_*` (новые, ниже) |

**Важно:** `n_tissues` = 20 — это ФИКСИРОВАННЫЙ скелет (базовые ткани), он НЕ
включает выросшие сайдкары. Не используй `n_tissues` как «сколько мозг
вырастил» — для этого `tissue_grown_live`.

## Источник данных

JSON диагностики (тот же, откуда UI берёт `topo_active`):
- **colony-агрегат** — top-level поля ответа диагностики;
- **per-creature** — объект Адама (`cid="c103927"`) в массиве creatures.

Оба уровня несут одинаковый набор `tissue_*` (для single-Adam агрегат = Адам).
Поля durable (переживают рестарт/сбой питания — персист в `.pt`).

## Поля контракта (эмитит клиент 0.13.52+)

| Поле | Тип | Смысл | Как показать |
|------|-----|-------|--------------|
| `tissue_grown_live` | int | ЖИВЫХ выросших тканей сейчас (вкл. 1 в GC-абляции) | **headline-число**: «Мозг вырастил N тканей» |
| `tissue_kept` | int | закреплено durable (= live; уменьшается при GC-пруне) | = live, можно не дублировать |
| `tissue_reverted` | int | отвергнуто ПРИ РОЖДЕНИИ (backoff: Δloss не значим за dwell) | мелким: «отклонено при росте: N» |
| `tissue_gc_pruned` | int | отпущено GC как noise-fit (вклад распался на полном погодном цикле) | мелким: «отпущено GC: N» |
| `tissue_propose_total` | int | всего предложено за жизнь (монотонный) | опц., «попыток: N» |
| `tissue_growing` | 0/1 | сейчас растит/оценивает НОВУЮ ткань (dwell) | индикатор-точка «растёт…» |
| `tissue_gc_evaluating` | 0/1 | сейчас GC-ре-оценка живой ткани (leave-one-out) | индикатор-точка «ревизия…» |
| `tissue_growth_enabled` | bool | флаг роста ON/OFF | бейдж ON/OFF |

### Stage 1 GRADUATION (клиент 0.13.53+): сайдкар → узел ГРАФА

Третья ступень: durable-ткань (GC-KEEP) «выпускается» из инкубатора сайдкаров
В ГРАФ организма (через cerebellum→motor, под §3-наблюдением). Это ДРУГОЕ
состояние ткани — graduated-узел НЕ входит в `tissue_grown_live`.

| Поле | Тип | Смысл | Как показать |
|------|-----|-------|--------------|
| `tissue_graduated_live` | int | узлов в ГРАФЕ (выпускники инкубатора) | «в графе: N» рядом с live |
| `tissue_grad_watch` | 0/1 | сейчас §3-watch graduated-узла (петля на паузе) | индикатор «● graduation…» |
| `tissue_grad_done` | int | успешных graduations за сессию | опц. |
| `tissue_grad_reverted` | int | revert'ов graduation (§3/energy-сигнал) | мелким |
| `tissue_graduation_enabled` | bool | флаг graduation ON/OFF | бейдж |

Семантика: graduation НЕ меняет `topo_active`-смысл (это рёбра скелетных
ролей); узел-выпускник добавляет СВОЁ ребро →cerebellum, но в UI он живёт
строкой «в графе: N». При revert узел возвращается в сайдкары
(`graduated_live`−1, `grown_live`+1) — это защита мотора, не регрессия.

Контекст-драйвер (поля уже есть, показывать рядом — рост ведёт ПРОГНОЗ):
- `loss_ema` / `prediction_loss_avg` — ошибка предсказания мира (драйвер);
- `prediction_accuracy` — точность прогноза;
- `intrinsic_ema` — внутренняя награда (любопытство), у floor = плато.

## Рекомендация по панели

```
┌─ Рост мозга ───────────────────────────────┐
│ Связи:  19/19 → cerebellum  (насыщено)      │   ← topo_active (стадия 1)
│ Ткани:  N выросших            ● растёт…     │   ← tissue_grown_live + флаги
│         отпущено GC: P · отклонено: R       │   ← gc_pruned · reverted
│ Прогноз: loss 0.0XX · точность 0.9X         │   ← драйвер
└─────────────────────────────────────────────┘
```

Состояния индикатора (приоритет сверху):
- `tissue_growth_enabled=false` → серый «OFF» (рост выключен);
- `tissue_gc_evaluating=1` → «● ревизия» (GC проверяет durability ткани);
- `tissue_growing=1` → «● растёт» (оценивает новую ткань);
- иначе → «idle» (ждёт плато прогноза / отдых между GC-эпохами).

## Семантика, чтобы UI не врал

- **«Выросло N»** — это `tissue_grown_live`, ЖИВЫЕ durable-ткани. Число может
  УМЕНЬШАТЬСЯ: GC отпускает ткани, чьё улучшение прогноза распалось на полном
  погодном цикле (фазовый шум, не структура). Падение `live` при росте
  `gc_pruned` — это НОРМА (чистка), не регрессия.
- `reverted` (backoff при рождении) и `gc_pruned` (отпущено позже ревизией) —
  РАЗНЫЕ события: первое = ткань не прижилась сразу, второе = прижилась, но
  durability не подтвердилась. Не суммировать в одно число.
- `live + gc_pruned` НЕ обязано = `propose_total` (часть в reverted, часть
  in-flight). Не выводить инвариант-арифметику.
- Рост ТКАНЕЙ ≠ рост СВЯЗЕЙ. Не складывать `tissue_grown_live` с `topo_active`.
